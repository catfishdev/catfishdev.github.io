You don't React

Some points

  • Build steps that take aeons to setup and run. Have you tried setting up your own Webpack config? Yeah not very cash money
  • VDOM and re-rendering the whole app to change contents of a label. And yeah I can wrap every single component in useMemo but aint nobody got time for that
  • Hooks and inconsistent state and props between renders. And yeah I can use refs or reducer but I want simple ways to do simple tasks
  • Enter the embedable XML syntax - JSX

    “Along with React, we are shipping an embeddable XML syntax. We’re calling that JSX.” pic.twitter.com/ONN8C0HC5G

    — Dan Abramov (@dan_abramov) December 7, 2019

So what to do instead?

Use a good ol server rendered app instead of building a SPA. Very rarely do you really need a SPA (pretty much never). Save time and sanity and be productive from day one. Go with either of these:

  • Laravel
  • Django
  • Ruby On Rails

But what about the front end? Well, to be honest, for a fairly simple app you don't even need a framework. Go vanilla. You have web components at your disposal for re-usable parts of the app. You have a single polyfill for older browsers that works.

  • No build step necessary
  • Older browser support
  • Dead simple API

But when building a SPA (you shouldn't), or when building a largely user interaction heavy application, what to use then?

Enter Svelte
  • Compiled to standard JavaScript
  • No VDOM
  • Dead simple API
  • Super easy to get started
  • Easily integrated with backend frameworks mentioned above
Or use whatever, this is just an opinion
